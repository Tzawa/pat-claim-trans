1. A memory controller for nonvolatile memory, the nonvolatile memory comprising one or more memory dies each having a set of blocks that must be erased before constituent storage locations of each of those blocks can individually be reprogrammed, the memory controller comprising:
logic to receive, from a host, write requests and read requests and to responsively issue commands to the nonvolatile memory to control fulfilment of the write requests and the read requests;
logic to store for each of the blocks in the set age information indicating when at least one of the constituent storage locations of the respective block was programmed;
logic to, from-time-to-time, and in a manner transparent to the host, control copying of data from a first one of the constituent storage locations to a second one of the constituent storage locations, in a manner where the copying is triggered dependent on the age information, and in a manner where the data also continues to reside in the first one of the constituent storage locations following the copying;
logic to track release information indicating that respective storage locations in the nonvolatile memory have been released, and to update said release information to identify each first one of the constituent storage locations as released in connection with copying of the data;
logic to identify at least one of the blocks in the set that is a candidate for erasure dependent on the tracked release information, and to transmit to the host a notification dependent on said identification; and
logic to, at a point subsequent in time to the transmission of the notification to the host, receive from the host an erase request that specifies one or more addresses associated with the nonvolatile memory, and to responsively control erasure of one or more of the blocks which corresponds to the one or more addresses, wherein the one or more addresses are effective to designate the first one of the constituent storage locations;
wherein each said logic comprises at least one of hardware circuitry or instructions stored on non-transitory machine-readable media that, when executed, are to control the operation of hardware circuitry.

2. The memory controller of claim 1, further comprising logic to update address translation information, and wherein the logic to receive the write requests and the read requests and to responsively issue the commands is to access said address translation information, so as to redirect an ensuing one of the read requests which is for the copied data to the second one of the constituent storage locations.

3. The memory controller of claim 1, wherein the memory controller further comprises logic to identify a condition where the age information indicates for one of the constituent storage locations that data has been stored at that one of the constituent storage locations for a period greater than a threshold amount of time, and wherein the logic to control the copying is to control copying in response to the identification of the condition.
